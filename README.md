# Shortlink — сервис коротких ссылок

Веб-приложение на Laravel для создания коротких ссылок, отслеживания
переходов и управления ссылками через личный кабинет на базе **Filament v3**.

## Стек

- Laravel 12 (соответствует требованию «Laravel 10+»)
- Filament v3 — аутентификация и личный кабинет пользователя целиком
  реализованы через Filament-панель (без отдельного пакета для auth)
- MySQL
- PHPUnit — feature-тесты

## Функциональность

- Регистрация и вход через Filament-панель (`/dashboard/register`,
  `/dashboard/login`).
- Создание короткой ссылки из произвольного URL.
- Редирект `GET /{shortCode}` → `original_url` с фиксацией перехода
  (IP-адрес, дата и время).
- Личный кабинет (`/dashboard/links`):
  - список своих ссылок с общим количеством переходов по каждой;
  - удаление ссылки;
  - детальный список переходов (IP, дата/время) внутри карточки ссылки.
- Каждый пользователь видит и может управлять только своими ссылками —
  ограничение реализовано и на уровне запроса в панели, и проверено тестами.

## Архитектурные решения

Несколько моментов, которые стоит отметить отдельно:

1. **302, а не 301 редирект.** Постоянный (301) редирект браузер закэширует
   на своей стороне, и повторные переходы по той же ссылке перестанут
   доходить до сервера — статистика будет занижена. Поэтому в
   `RedirectController` используется временный (302) редирект.

2. **Количество переходов считается на лету (`withCount('clicks')`)**,
   а не хранится в отдельном поле. Рассматривался вариант с денормализацией
   (отдельная колонка `clicks_count`, обновляемая через `increment()`), но
   для объёма данных тестового задания это избыточно и добавляет риск
   рассинхронизации счётчика с реальными данными, если где-то забыть
   вызвать инкремент. `withCount()` даёт тот же результат одним
   дополнительным подзапросом, без отдельного состояния для поддержки.

3. **Короткий код хранится отдельно от домена.** В таблице `links` хранится
   только `short_code` (например, `abc123`), а полная короткая ссылка
   собирается на лету через accessor `short_url` в модели `Link`
   (`url('/'.$this->short_code)`). Если проект переедет на другой домен —
   все ссылки останутся рабочими, ничего не нужно мигрировать.

## Установка

```bash
git clone https://github.com/n3show/shortlink.git
cd shortlink

composer install

cp .env.example .env
php artisan key:generate

# В .env указать доступ к MySQL, например:
# DB_CONNECTION=mysql
# DB_HOST=127.0.0.1
# DB_PORT=3306
# DB_DATABASE=shortlink
# DB_USERNAME=root
# DB_PASSWORD=
# Базу данных shortlink нужно создать заранее (например, через phpMyAdmin/CLI)

php artisan migrate

php artisan serve
```

Приложение будет доступно на `http://localhost:8000`, личный кабинет — на
`http://localhost:8000/dashboard`. Первого пользователя можно создать через
`http://localhost:8000/dashboard/register`.

## Тесты

```bash
php artisan test
```

Покрыты: редирект на оригинальный URL, фиксация перехода в статистике при
одном и при нескольких переходах подряд, 404 для несуществующего кода,
запрет доступа в личный кабинет для неавторизованных пользователей.
